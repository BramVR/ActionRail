# Vision

ActionRail is a polished, data-driven Maya viewport UI system for artists and TDs. It should keep deepening the PySide6/Qt overlay experience, authoring workflow, presets, and Maya-native verification while avoiding premature Viewport 2.0 or broad tool-suite scope.

## Merge by Default

- Bug fixes with clear cause, bounded risk, and Maya or test proof.
- Current-phase implementation work that matches `docs/02_implementation_plan.md`.
- Small UI, preset, diagnostics, and authoring improvements that preserve the declarative rail model.
- Documentation and project-map updates that keep agent handoff, status, and verification routes current.
- Tests, smoke checks, and MayaSessiond workflow improvements.

## Needs Sign-Off

- New phases beyond the active roadmap step.
- Viewport 2.0, PyMEL, custom renderer, or large framework changes.
- Bind Mode, flyouts, command rings, profile layers, or marking-menu export before their planned phase.
- Changes that mutate locked bundled presets or blur bundled versus user preset ownership.
- Behavior that cannot be verified in Maya when the changed surface is Maya runtime behavior.
